## コンテキスト

- カテゴリ: !`cat .claude/CURRENT_CATEGORY 2>/dev/null || echo app`
- 最新バージョン番号: !`bash .claude/scripts/get_latest_version.sh`

## 対応

最新バージョンの `MEMO.md` を参照して、記載された各項目に対応してください。

### MEMO に対応事項がない場合

`MEMO.md` の内容が「特になし」等で対応すべき項目がない場合は、以下の軽量チェックのみ行い、結果をユーザーに報告して完了とする:

1. `npx nuxi typecheck` で型エラーがないことを確認
2. 未使用の import や変数がないか簡易確認（サブエージェントで調査）
3. ISSUES 整理（下記「進め方」の step 3 と同じ手順で実施）
4. 結果をユーザーに報告して完了とする

### 対応方針

各 MEMO の記載に対して、以下のいずれかの対応を取ってください:

- ✅ **対応完了**: 修正内容を具体的に記載
- ⏭️ **対応不要**: 理由を明確に記載し、**必ずplan_review_agentサブエージェントに報告して承認を得ること**
- 📋 **次バージョンへ先送り**: スコープ外である理由を記載し、plan_review_agentサブエージェントに報告。**先送りにした内容は `ISSUES/{カテゴリ}` フォルダに追加すること**（優先度に応じて `high` / `medium` / `low` に配置）

### 進め方

1. `MEMO.md` の各項目について、上記いずれの対応を取るかplan_review_agentサブエージェントを起動して確認・質問を積極的に行う
2. 全ての項目への対応が完了したら、 `MEMO.md` を更新して対応結果を記載する
3. ISSUES整理
   - 解決された `ISSUES/{カテゴリ}` フォルダ配下の課題をファイルごと削除
   - 修正適用済みだが外部検証待ち（デプロイ後の確認等）の ISSUES には、ファイル末尾に `## ステータス` セクションを追記して現状を記録する（例: `修正適用済み（ver2.1）、デプロイ検証待ち`）
   - `ISSUES/{カテゴリ}` フォルダに追加する内容をユーザーに提案して、ユーザーの指示に従って追加する
4. 最終的な対応結果の一覧をユーザーに報告する
5. Git にコミットする
- wrap_up で行った変更（コード修正・MEMO.md 更新・ISSUES 整理等）をコミットする
- コミットメッセージ例: `docs(ver{バージョン番号}): wrap_up完了`
- **コミット対象の変更が存在しない場合**（MEMO に対応事項なし・ISSUES 変更なし等）は、コミットをスキップしてよい。その旨を `MEMO.md` 末尾または進捗報告に「wrap_up: 変更なし（スキップ）」と明記してから次ステップに進む
