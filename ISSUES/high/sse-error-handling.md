# SSE error イベントのクライアント処理

## 概要

サーバーは `token` / `done` / `error` の3種のSSEイベントを送信するが、クライアント (`index.vue`) は `token` のみ処理している。
ストリーム途中のエラー（OpenAI APIレート制限など）がユーザーに通知されない。

## 現状の問題

- `error` イベントが送信されても、クライアントは無視してストリーム終了を待つだけ
- ユーザーにはエラーの原因が伝わらず「応答を取得できませんでした」と表示される

## 対応方針

- クライアントのSSEパーサーで `error` イベントを検知し、エラーメッセージをユーザーに表示する
- `done` イベントも明示的に処理してストリーム完了を確定させる
