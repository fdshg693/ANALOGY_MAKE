# スキルファイル改善リクエスト（ver14.3 retrospective）

retrospective で以下2つのスキル改善を提案しましたが、`.claude/skills/` への書き込み権限がないため適用できませんでした。
ユーザーによる手動適用をお願いします。

## 改善1: split_plan — 小規模タスクの IMPLEMENT.md 簡潔化

**対象**: `.claude/skills/split_plan/SKILL.md`

**変更箇所**: ステップ2 の IMPLEMENT.md の説明部分

**変更前**:
```
- `IMPLEMENT.md` は通常通り作成する
```
（小規模タスクセクション内）

および:
```
   - `IMPLEMENT.md` — タスクの実装案の作成。可能な限り、実装の詳細に踏み込むこと。新規ライブラリや未使用APIを扱う場合は、「## リスク・不確実性」セクションを設け、型定義の不備・ドキュメント不足・実行時挙動の不確実性を洗い出すこと
```

**変更後**:
```
- `IMPLEMENT.md` は作成するが、下記の簡潔化ガイドラインに従う
```

および:
```
   - `IMPLEMENT.md` — タスクの実装案の作成。新規ライブラリや未使用APIを扱う場合は、「## リスク・不確実性」セクションを設け、型定義の不備・ドキュメント不足・実行時挙動の不確実性を洗い出すこと
     - **通常タスク**: 可能な限り実装の詳細に踏み込むこと
     - **小規模タスク**: 記述量をコード変更量の2〜3倍程度に抑えること。核心の変更内容とその理由に絞り、網羅的な代替案の列挙や詳細なフォールバック計画は省略してよい
```

**理由**: ver14.3 で実際のコード変更が ~15行であるのに対し、IMPLEMENT.md が207行と過剰だった。

## 改善2: imple_plan — 段階的アプローチのスキップ条件明示

**対象**: `.claude/skills/imple_plan/SKILL.md`

**変更箇所**: 実装品質ガイドラインの「計画との乖離」の直後に追加

**追加内容**:
```
- **段階的アプローチのスキップ**: IMPLEMENT.md に診断→修正など段階的アプローチが計画されている場合、以下の条件をすべて満たせば前段階をスキップして直接修正してよい（スキップ理由は `MEMO.md` に記載）:
  - 修正が前段階の結果に関わらず安全である（上位互換の手法への置換など）
  - 前段階の実施にサーバー起動やブラウザ操作など対話的な手順が必要
  - 仮説の蓋然性が十分に高い（既知のパターン、公式ドキュメントに記載など）
```

**理由**: ver14.3 でフェーズ1（診断）のスキップが妥当だったが、スキップ判断の基準がスキルに明示されていなかった。
