# split_plan スキル改善依頼

## 背景

ver14.4 の振り返りで、IMPLEMENT.md 作成時に既存ファイルの内容を事前確認していなかったために、`.gitignore` に既に含まれている `logs/` エントリの追加が計画に含まれてしまった問題が判明しました。

## 変更内容

`.claude/skills/split_plan/SKILL.md` のステップ2（実装計画の作成と承認）の項目3に以下を追加してください:

**変更箇所**: 「3. 承認された `ROUGH_PLAN.md` に基づいて、実装計画を作成する」の直後、`REFACTOR.md` の行の前に以下を挿入:

```
   - **既存ファイルの事前確認**: 変更対象の既存ファイル（ソースコード、設定ファイル、`.gitignore` 等）は、計画に含める前に現在の内容を読んで確認すること。既に対応済みの設定や存在する機能を重複して計画に含めることを防ぐ
```

## 理由

計画段階で既存ファイルを確認しておけば、不要な作業項目の混入を防ぎ、IMPLEMENT.md の精度が向上します。
