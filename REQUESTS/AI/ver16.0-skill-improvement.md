# ver16.0 retrospective スキル改善（権限ブロックのため未適用）

## 概要

`/retrospective` 実行中に `.claude/skills/imple_plan/SKILL.md` への即時編集を試みたが、権限プロンプトが表示され AUTO モードでは承認できず適用できなかった。ユーザー承認後に以下の差分を適用されたい。

## 差分

`.claude/skills/imple_plan/SKILL.md` の「動作確認」セクション、リスク・不確実性の処理リストの「検証先送り」行を次の内容に置き換える:

### 現在

```
   - 検証先送り → 先送り理由と、本番発生時の対応方針を記載
   - 未記載のまま完了するとリスクが放置されるので必ず触れること
```

### 置換後

```
   - 検証先送り → 先送り理由と、本番発生時の対応方針を記載。**加えて `ISSUES/{カテゴリ}/{優先度}/{短い名前}.md` に独立したファイルを作成する**（概要・本番発生時の兆候・対応方針・影響範囲を記載）。これにより先送りリスクが MEMO 内に埋もれて次バージョンで見落とされるのを防ぐ。MEMO 側には `ISSUES/.../xxx.md に追加済み` と参照を残す
   - 未記載のまま完了するとリスクが放置されるので必ず触れること
```

## 背景

ver16.0 で「検証先送りリスクを ISSUES 化する」パターンが自発的に適用され、リスクが MEMO 内に埋もれる問題を実質的に解消した（`ISSUES/app/medium/getState-timing.md`, `additional-kwargs-sqlite.md`）。このパターンをスキル本体に恒久反映することで、次バージョン以降も再現可能にする。

詳細は `docs/app/ver16.0/RETROSPECTIVE.md` 4-A 項を参照。
