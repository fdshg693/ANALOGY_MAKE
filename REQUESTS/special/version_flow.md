# 現状のVersion記載の変更

## 現状

ver1 -> ver2 -> ver3 -> ver4

- メジャーバージョン等の区切りがないため、チェックポイントとして `CURRENT.md` を作成する際に、その時の現状を全て詰め込んだ、重複の多い内容になっている。

## 変更案

ver 1.0 -> ver 1.1 -> ver 1.2 -> ver 2.0

- メジャーバージョンは、実装ワークフローの大きな変更があったときに上げる。
  - `CURRENT.md` を作成する際に、 ver\*.0 のみ現状を反映した完全版を記述して、マイナーバージョンアップの際には、変更点のみを記述する。
