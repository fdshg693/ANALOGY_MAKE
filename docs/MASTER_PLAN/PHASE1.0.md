# PHASE1: アナロジー思考AIアシスタント最小限実装

## コンセプト

ユーザーが抱える課題に対し、**アナロジー思考**（異分野の類似事例から着想を得る手法）をAIが補助するチャットアプリ。
課題の抽象化 → 他分野の類似事例の提示 → ユーザー選択 → 解決策の提案、という一連のフローを対話形式で行う。

## 対話フロー

```
1. ユーザーが課題を入力
   例: 「新幹線がトンネルに入る時に音がドンとなる」

2. AIが課題を抽象化して提示
   例: 「物体が衝突するときに、衝撃を発生させない」

3. AIが抽象化した概念に類似する他分野の事例を複数提示
   例:
   - カワセミが水に入る時に波が立たない
   - 潜水艦の静音航行技術
   - ...

4. ユーザーが気になる事例を選択（または自由に指示）
   例: 「カワセミの事例と組み合わせて」

5. AIが選択された事例を元に解決策を提案
   例: 「新幹線の先頭形状をカワセミの嘴の形にする」
```

ユーザーは任意のステップで自由に追加質問・やり直しができる。

## やること

- 1画面のシンプルなチャットUI（入力欄 + 会話履歴）
- 上記5ステップの対話フローをLLMのプロンプト設計で実現
- LLM（OpenAI API）の内部知識のみで類似事例を生成
- LangChainでプロンプトチェーンを構築
- NuxtJSによるフロント + APIサーバー
- CLIからも同じロジックを実行可能にする

## やらないこと

- 認証・ユーザー管理
- 会話履歴の永続化（DBへの保存）
- 事例データベースの構築（RAG構成）
- Web検索による最新事例の取得
- 複数会話セッションの管理
- レスポンシブ対応・モバイル最適化
- デプロイ・本番運用

## 構成の全体像

```
[ブラウザ]
  └── NuxtJS (フロントエンド)
        ├── チャットUI (1画面)
        └── API Route (サーバーサイド)
              └── LangChain
                    └── OpenAI API

[CLI]
  └── LangChain (同じロジックを共有)
        └── OpenAI API
```

- **フロントエンド**: NuxtJSのページ1枚。チャットの送受信のみ
- **バックエンド**: NuxtJSのAPI Routeとして実装。アナロジー思考のロジックはここに集約
- **AI連携**: LangChainでプロンプトチェーンを組み、OpenAI APIを呼び出す
- **CLI**: バックエンドのロジックを直接呼び出すスクリプト。高速な実験用
