# PHASE1.5: 品質改善・リファクタリング（PHASE1からの差分）

## 概要

PHASE2（永続化・マルチスレッド・Web検索・ステートマシン化）に備え、
PHASE1で積み残した技術的負債を解消し、コードベースの拡張性と保守性を確保する。

主に以下の4点を対応する:
1. **フロントエンドのリファクタリング** — index.vue からロジックを composable + ユーティリティに分離
2. **応答キャンセル機能** — ストリーミング中の中断手段を提供
3. **ストリーミング表示の改善** — トークン表示の滑らかさとMarkdownパースの安定化
4. **自動テストの導入** — Vitestによるリグレッション検知の安全網

## やること

### 1. composable + SSEパーサー切り出し

`index.vue`（183行）にSSEパース・エラーハンドリング・UI状態管理が同居しており、
PHASE2でスレッド切り替えUIを追加する際に肥大化が避けられない。ロジックを分離する。

- SSEパース処理を `app/utils/sse-parser.ts` に切り出し
  - ReadableStream を消費して token / done / error イベントをコールバックで通知する関数
  - Vueに依存しない純粋関数として実装
- メッセージ管理・送信・ストリーミング状態を `app/composables/useChat.ts` に切り出し
  - リアクティブ状態: `messages`, `isLoading`, `isStreaming`, `threadId`
  - `sendMessage()`: fetch + SSEパーサー呼び出し + エラーハンドリング
- `index.vue` には auto-scroll の watch とテンプレートのみ残す（約60–70行に縮小）

### 2. 応答キャンセル機能

ストリーミング中にユーザーが応答を中断する手段がない。
長い応答や意図しない方向の応答を途中で止められるようにする。

- `useChat` に `abort()` 関数を追加（`AbortController` で fetch を中断）
- 中断時の部分テキストは保持（削除しない・エラーにもしない）
- `ChatInput` のボタンをストリーミング中は「停止」に切り替え

### 3. ストリーミング表示の改善

トークンが大きなチャンク単位でしか更新されず、なめらかに表示されない問題を改善する。
また、ストリーミング中の不完全なMarkdownの表示崩れを軽減する。

- バッファリングの原因調査（ブラウザ側 or サーバー側）
- 必要に応じてサーバーレスポンスヘッダーの調整
- ストリーミング中のMarkdownパースの安定化（末尾補完等）

### 4. 自動テスト導入（Vitest）

手動確認に完全依存している現状を改善し、PHASE2の大規模変更前にリグレッション検知の安全網を整備する。

- Vitest + happy-dom のセットアップ
- SSEパーサーのユニットテスト（バッファ分割・中断等のエッジケース）
- useChat composable のテスト（状態遷移・エラーハンドリング）
- サーバーAPIハンドラのテスト（バリデーション・イベントストリーム）
- 外部API（OpenAI）はモックし、APIキーなしで実行可能にする

## やらないこと

- 認証・ユーザー管理
- 会話履歴の永続化（PHASE2で対応）
- 複数会話セッションの管理（PHASE2で対応）
- 事例データベースの構築（RAG構成）
- Web検索による最新事例の取得（PHASE2で対応）
- プロンプトへのカテゴリ追加（PHASE2でプロンプト再設計時に対応）
- シンタックスハイライト（アナロジー対話でコードブロックは稀、優先度最低）
- モバイル最適化
- デプロイ・本番運用

## 構成の変更点

```
PHASE1:
  app/pages/index.vue (183行、全ロジック内包)
    └── fetch → server/api/chat.post.ts → LangChain → OpenAI API

PHASE1.5:
  app/pages/index.vue (60-70行、UIのみ)
    └── app/composables/useChat.ts (状態管理・送信・中断)
          └── app/utils/sse-parser.ts (SSEパース)
                └── fetch → server/api/chat.post.ts → LangChain → OpenAI API

  tests/
    ├── utils/sse-parser.test.ts
    ├── composables/useChat.test.ts
    └── server/chat.test.ts
```
