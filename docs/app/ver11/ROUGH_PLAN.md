# ver11 タスク概要: 複数スレッド管理

## 対応方針

MASTER_PLAN PHASE2 の次項目「2. 複数スレッド管理」を実装する。
`ISSUES/app/high/` に未解決の課題はないため、MASTER_PLAN に沿った機能追加を進める。

## 背景

ver10 で SQLite による会話履歴永続化が完成し、サーバー再起動後も会話を継続可能になった。しかし現状では localStorage に保存された単一の threadId しか扱えず、ユーザーは1つの会話しか保持できない。

## ver11 で提供する機能

ユーザーが複数の会話スレッドを作成・切り替え・管理できるようにする。

### 機能一覧

1. **サイドバーにスレッド一覧を表示**
   - 画面左側にサイドバーを配置し、既存のスレッド一覧を表示する
   - 現在アクティブなスレッドを視覚的に区別する

2. **スレッド一覧取得APIの追加**
   - 現状の LangGraph SqliteSaver にはスレッド一覧を返す機能がないため、スレッドのメタデータ（threadId・タイトル・作成日時）を管理する仕組みを新たに用意する
   - スレッド一覧を返す新規APIエンドポイントを追加する

3. **「新規チャット」ボタン**
   - サイドバー上部に配置し、新しい空のスレッドを作成する
   - 作成後は自動的にそのスレッドに切り替わる

4. **スレッドタイトルの自動生成**
   - AIの最初の返答完了後に、バックエンド側で会話内容からタイトルを自動生成する
   - タイトル生成は既存の `POST /api/chat` の処理内で非同期に実行し、専用エンドポイントは設けない
   - タイトル未生成の新規スレッドは「新しいチャット」等の仮タイトルで表示する

5. **スレッド切り替え**
   - サイドバーのスレッドをクリックすると、そのスレッドの会話履歴をロードして表示する
   - 切り替え時に現在の会話状態を適切にリセットする

### 事前リファクタリング不要

現在のコード構造（`useChat.ts` 124行、バックエンドは threadId 別管理が既に動作）には構造的なブロッカーがなく、機能追加と新規ファイル作成で対応可能。

### スコープ外（ver11 では対応しない）

- スレッドの削除機能
- スレッドの並び替え・検索
- スレッドのピン留め・アーカイブ
- サイドバーの開閉トグル（常時表示）
