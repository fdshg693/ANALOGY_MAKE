# ver13.1 タスク概要: サーバーサイドログの追加

## 対応する課題

`ISSUES/app/high/logging.md` — コア機能の動作確認ログが不在

## 背景

現在、サーバーサイドのコード全体にログが一切存在しない。特に ver12 で追加された Tavily Search が実際に実行されているかどうかを確認する手段がなく、動作検証が困難な状態にある。

## タスクの内容

サーバーサイドのコア機能に対し、最低限の動作確認ができる粒度のログを追加する。

### ログ対象

1. **エージェント初期化** — モデル名、ツール構成（Tavily 有効/無効）、チェックポイント DB パスの出力
2. **チャット API** — リクエスト受信、ストリーミング開始/完了/エラー、ツール呼び出し検出
3. **タイトル自動生成** — 生成開始/完了/エラー
4. **DB 操作** — DB 初期化、スレッド作成/更新
5. **会話履歴取得** — リクエスト受信、取得結果のメッセージ数、エラー（現在サイレントに失敗している箇所の可視化）

### スコープ外

- 外部ログライブラリの導入（console ベースで十分）
- ログローテーション・ログレベル設定
- クライアントサイドのログ
- パフォーマンスメトリクス

## バージョン種別

マイナーバージョン（ver13.1）: 既存機能への改善、新規ライブラリ不要、アーキテクチャ変更なし

## 事前リファクタリング不要

ログ追加は既存コードへの追記が主であり、構造変更を伴わないためリファクタリングは不要。
