# ver14.0 タスク概要: LangGraphステートマシンによるフロー制御

## 対応する MASTER_PLAN 項目

`docs/app/MASTER_PLAN/PHASE2.md` — 項目4「LangGraphステートマシンによるフロー制御」

## 背景

現在のアナロジー思考フローは、単一の `createReactAgent` と1つのシステムプロンプト（`ANALOGY_SYSTEM_PROMPT`）により、5ステップすべてをLLMのプロンプト制御に委ねている。`createReactAgent` は内部で既に LangGraph の `CompiledStateGraph` を使用しているが（`history.get.ts` の `getState()` が動作していることがその証拠）、今回はこの「暗黙の LangGraph 利用」を「明示的な StateGraph 定義」に置き換える作業である。

現方式の課題:

- ステップの実行順序がLLMの判断に依存し、再現性が低い
- ステップ3（類似事例提示）でのWeb検索実行が「推奨」止まりで、強制できない
- 各ステップの入出力が不明瞭で、デバッグや改善が困難

LangGraphステートマシンに移行することで、各ステップを独立したノードとして明示的に分離し、フローの再現性と制御性を大幅に向上させる。

## タスクの内容

現在の単一ReActエージェントを、LangGraph StateGraph によるマルチノード構成に置き換える。

### ノード構成（3ノード）

MASTER_PLAN では「事例検索ノード」と「事例提示ノード」を分離しているが、事例検索の出力をそのままフォーマットして提示すれば足りるため、これらを統合した3ノード構成とする:

1. **抽象化ノード**: ユーザーの課題を受け取り、抽象的な概念に変換する。ステップ2に相当
2. **事例検索・提示ノード**: 抽象化結果を元に、LLM内部知識 + Tavily Searchで類似事例を収集し、フォーマットしてユーザーに提示する。ステップ3に相当。Web検索をノードロジックとして組み込むことで、プロンプト依存から脱却する
3. **解決策生成ノード**: ユーザーが選択した事例に基づき、具体的な解決策を提案する。ステップ5に相当

### interrupt による中断・再開フロー

LangGraphの `interrupt` 機構を使い、事例提示後にグラフを一時停止する:

- **初回メッセージ（課題入力）**: グラフが開始し、抽象化 → 事例検索・提示を実行。提示後に `interrupt` でグラフを中断
- **2回目のメッセージ（事例選択）**: `Command({ resume: userMessage })` でグラフを再開し、解決策生成ノードを実行
- **API契約の維持**: `POST /api/chat` の `{ message, threadId }` リクエスト形式は変更しない。フロントエンドはユーザーのテキストメッセージをそのまま送信し、サーバー側でグラフの状態に応じて `stream()` か `Command` による再開かを判断する

### 既存API・履歴取得の互換性

- **`history.get.ts`**: StateGraph 移行によりステートのキー構造が変わる可能性がある。`messages` に加え、`currentStep`、`abstractedProblem` 等のカスタムキーがステートに追加されるため、`getState()` の返り値からメッセージを抽出するロジックを更新する
- **ストリーミングフィルタ**: `chat.post.ts` の `AIMessageChunk` フィルタリングは、StateGraph のストリーミングモード（`streamMode` の選択）に応じて見直す。ノード出力をストリーミングする場合、フィルタロジックが変わる可能性がある

### テスト方針

アーキテクチャの根本変更に伴い、`tests/server/chat.test.ts` と `tests/server/chat-history.test.ts` のモック構造を新しいグラフ構成に合わせて改修する。詳細は IMPLEMENT.md で定義する。

### ユーザー体験の変化

- 対話フロー自体は従来と同じ5ステップを維持（UIに変更なし）
- ステップ3が確実にWeb検索を含むようになり、事例の質と鮮度が向上する
- 各ステップの境界が明確になるため、将来的にステップごとのUI表示（プログレス表示等）が可能になる

## スコープ

変更対象はサーバーサイドのみ（`server/` 配下）。フロントエンド（`app/` 配下）への変更は不要。

主な変更ファイル:
- `server/utils/analogy-agent.ts` — StateGraph 定義に全面書き換え
- `server/utils/analogy-prompt.ts` — ノード別プロンプトへの分割
- `server/api/chat.post.ts` — グラフ実行・ストリーミング処理の更新
- `server/api/chat/history.get.ts` — ステート構造変更への対応

## 事前リファクタリング不要

`analogy-agent.ts` は全面書き換え、他ファイルも対象箇所が明確であるため、事前のリファクタリングステップは不要。
