# ver14.2 ROUGH_PLAN — ストリーミング表示のレンダリング最適化

## 対応 ISSUE

`ISSUES/app/low/streaming.md` — ストリーミング中の表示改善

## タスク概要

ストリーミング中の Markdown レンダリングを最適化し、滑らかな表示を実現する。

### 現状の問題

1. **毎トークンごとの全文再解析**: ストリーミング中、トークンを1つ受信するたびに `marked.parse()` + `DOMPurify.sanitize()` が全文に対して実行される。500トークンのレスポンスでは500回の全文解析が走り、後半になるほどレンダリングが重くなる
2. **不完全 Markdown の表示崩れ**: ストリーミング中は Markdown が不完全な状態（未閉じのコードブロック、途中のテーブルなど）で `marked.parse()` に渡されるため、一瞬レイアウトが崩れて見える可能性がある

### 実現する改善

- ストリーミング中の Markdown レンダリング頻度を間引き、滑らかかつ軽量な表示を実現する
- ストリーミング完了時には即座に完全な Markdown レンダリングを行い、最終表示の品質を保証する
- ユーザーの体感として「文字がスムーズに流れてくる」印象を実現する

### スコープ

- フロントエンドのレンダリング最適化のみ（サーバー側の SSE 送信ロジックは変更しない）
- SSE パーサー（`sse-parser.ts`）も変更しない
- 新規外部ライブラリの導入なし

### 事前リファクタリング不要

既存コードの構造はそのまま活用できる。ChatMessage.vue のレンダリングロジック変更と index.vue の props 追加のみで実現可能であり、構造的なリファクタリングは不要。

### スコープ外

- サーバー側のストリーミングチャンク粒度の調整（LangGraph の内部挙動に依存）
- Markdown パーサーライブラリの変更（marked は継続使用）
