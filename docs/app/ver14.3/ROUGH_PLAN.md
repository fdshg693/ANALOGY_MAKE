# ver14.3 ROUGH_PLAN — 履歴不具合の修正

## 対応 ISSUE

- `ISSUES/app/high/履歴不具合.md`
- ブラウザ再読み込みやアプリ再起動後に、AI側のメッセージ履歴が消失する（人間側のメッセージは正常に復元される）
- ローカル・Azure 両方で発生

## 問題の概要

チャット履歴は LangGraph の `SqliteSaver` チェックポイントに依存して永続化されている。ブラウザリロード時に `GET /api/chat/history` が `agent.getState()` でチェックポイントから状態を取得するが、AI メッセージが保存されていないため復元できない。

人間メッセージは復元されるが AI メッセージは復元されないことから、チェックポイントへの保存または取得のいずれかの段階に問題がある可能性がある。原因の詳細は実装フェーズで調査する。

## ゴール

- ブラウザ再読み込み・アプリ再起動後も、人間・AI 双方のメッセージ履歴が正しく復元されること
- ローカル・Azure 両環境で動作すること
- 既存のストリーミング表示体験（ver14.2 で最適化済み）を損なわないこと

## スコープ

- 履歴永続化のバグ修正のみ
- 小規模タスクのため REFACTOR 省略（変更対象は主にサーバーサイドの履歴保存・取得ロジック）

## スコープ外

- フロントエンドの UI 変更
- 新機能追加
- 他の ISSUE 対応
