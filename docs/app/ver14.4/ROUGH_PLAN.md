# ver14.4 ROUGH_PLAN

## 対応ISSUE

`ISSUES/app/high/履歴修正-実機確認.md`

ver14.3 で `instanceof` → `isInstance` への置き換えを実施したが、履歴不具合が継続している。
根本原因の調査を可能にするための **デバッグ・調査ツール** を整備する。

## 目的

履歴不具合の原因を特定・検証できる環境を作る。バグ修正そのものは本バージョンのスコープ外とし、調査基盤の構築に集中する。

## 提供する機能

### 1. 永続ログ機能（ローカル環境限定）

チャットAPI処理の要所（メッセージ保存・履歴取得・スレッド操作）でログを `logs/` フォルダに出力する仕組みを追加する。ログは開発環境でのみ有効化し、本番環境では無効とする。

ログにより、以下が後から確認可能になる:
- APIリクエスト/レスポンスの内容
- LangGraph チェックポイントへの保存内容
- 履歴取得時のデータ復元過程

### 2. SQLite 調査CLIツール

SQLiteデータベースの内容を直接確認できるCLIツールを作成する。対話的なUIは不要で、コマンドライン引数で操作を指定する形式で十分。

主な機能:
- スレッド一覧の表示
- 指定スレッドのチェックポイント（メッセージ履歴）の表示
- メッセージ内容のデコード・整形表示

## スコープ外

- 履歴不具合の修正そのもの（調査結果に基づく修正は次バージョン以降）
- 本番環境へのログ機能デプロイ
- GUIベースの管理ツール
