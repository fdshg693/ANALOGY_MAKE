# ver15.0 ROUGH_PLAN — 動的設定システム & AI回答粒度切り替え

## 対象

MASTER_PLAN PHASE3 の項目1「動的設定システム」および項目1.1「AI回答粒度設定」に着手する。

## 前提条件

- ver14.5 で修正した履歴フィルタリング（`isInstance` → `type` プロパティベース）の実機確認が完了していること

## 背景

PHASE2 までで、アナロジー思考の5ステップ対話フロー（抽象化 → 事例検索 → 事例提示 → 選択 → 解決策提案）と、会話履歴・スレッド管理・Web検索連携・LangGraph StateGraph によるフロー制御が完成している。

現状、AI の回答スタイルは固定のシステムプロンプトで決定されており、ユーザーが調整する手段がない。簡潔な回答が欲しい場合も、詳細な説明が欲しい場合も、同じプロンプトで動作する。

## 今回のスコープ

### 提供する機能

1. **スレッドごとの設定保存基盤**: 各スレッドに設定値（JSON）を紐づけて永続化する仕組み
2. **AI回答粒度の切り替え**: 「簡潔」「標準」「詳細」の3プリセット + 自由テキストによるカスタム指示
3. **設定パネルUI**: チャットエリア上部にトグル表示する設定パネル

### ユーザー体験の変化

- チャット画面に設定ボタン（歯車アイコン等）が追加される
- 設定パネルを開くと、回答粒度のプリセット選択（簡潔/標準/詳細）とカスタム指示入力が表示される
- 設定変更は即座に次の AI 応答から反映される（過去の応答は変わらない）
- 設定はスレッドごとに保持され、他のスレッドに影響しない

### スコープ外（後続バージョンで対応）

- 検索パラメータの動的切り替え（PHASE3 項目1.2）
- 検索結果の可視化（PHASE3 項目2）
- 会話分岐（PHASE3 項目3）
- エコーモード・システムプロンプトカスタマイズ（PHASE3 項目4）
