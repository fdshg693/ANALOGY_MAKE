### 実装内容

ver15.0 の動的設定システムを拡張し、Tavily Web 検索のパラメータ（ON/OFF・検索深度・取得件数）をスレッドごとに切り替え可能にした。

1. **型拡張**: `ThreadSettings.search: SearchSettings` 追加、`DEFAULT_SEARCH_SETTINGS` エクスポート
2. **後方互換マージ**: `getThreadSettings()` を段階マージに変更（旧レコードへの `search` 欠損対応）
3. **実装**: `performSearch(query, search)` シグネチャ変更 → Tavily コンストラクタへ `searchDepth`・`maxResults` 動的注入
4. **バリデーション**: PUT API で `enabled` の型チェック・`depth` ホワイトリスト・`maxResults` の 1〜10 クランプ
5. **UI**: `SettingsPanel.vue` に検索設定セクション（トグル + 2 セレクト、OFF 時は disabled）
6. **テスト**: `settings-api.test.ts` に 6 ケース、`thread-settings.test.ts` に 2 ケース、`chat.test.ts` の型追従更新

## 3. 次バージョンの種別推奨

### 推奨: メジャーバージョン 16.0

**対象**: PHASE3 項目 2「検索結果の可視化（Tavily 検索の生データ表示）」

**理由**:

1. **新機能のアーキテクチャ追加**: 現状 `caseSearchNode` は検索結果を LLM プロンプトに埋めるだけで、クライアントには届かない。可視化には SSE イベント新設（`search_results` または類似）と、フロント側の表示コンポーネント新設が必要
2. **データフロー拡張**: 検索結果の構造化パース（JSON 文字列 → 型付きオブジェクト → SSE → UI）という新しいレイヤーが加わる。ver15.1 MEMO.md の「潜在的な将来対応」でも関連事項として記録されている
3. **破壊的ではないが新しい機能の本体**: MASTER_PLAN PHASE3 の新項目に着手する、というメジャー条件に該当
