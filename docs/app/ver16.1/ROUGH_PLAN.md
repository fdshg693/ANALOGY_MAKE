## 目的

開発者（ローカル動作確認時）の負担を減らす。現状は少しの UI 確認でも AI が正式な長文回答を返し、時間とトークンコストがかかる。以下 2 つのスイッチで、チャット UI そのものの挙動と AI 本体の挙動を切り離して確認できるようにする。

## スコープ

### 1. エコーモード

**ユーザー体験:**

- 設定パネルに「応答モード」を追加。選択肢は `AI`（デフォルト）/ `エコー` の 2 択
- エコーモード中は LangGraph エージェントを完全にバイパスし、サーバーがユーザー入力をそのまま assistant メッセージとして SSE で返す
- トークンストリーミングの表示確認を成立させるため、入力テキストを小分けにして配信する（詳細な分割方法は IMPLEMENT で決定）
- 会話履歴には保存する（通常フローと同様に永続化）。履歴再読み込み時の表示も通常メッセージと同じ
- エコーモードでは Web 検索は必ず OFF 相当（`search_results` イベントは送信しない）

**対象外（このバージョンではやらない）:**

- エコー応答に意図的な遅延・エラー注入などの挙動テスト用オプション
- エコーモード専用の別エンドポイント追加（既存 `/api/chat` の分岐で実現する）

### 2. システムプロンプト上書き

**ユーザー体験:**

- 設定パネルに「システムプロンプト上書き」テキストエリアを追加
- 入力値は各ノード（abstraction / caseSearch / solution / followUp）の既存システムプロンプトの**先頭に追記**される
- 空文字のときは何もしない（現行動作）
- 上書きは**開発環境でのみ表示・有効**。本番では UI も効果も無効（常に空扱い）
- スレッドごとに保持（既存 `ThreadSettings` と同じスコープ）

**対象外:**

- ノード単位でプロンプトを差し替える機能（今回は「前置き」のみ）
- 複数プリセットの保存・切り替え UI
- 本番環境で上書きを有効にするオプション

### 3. 設定パネル UI への統合

- 既存 `SettingsPanel.vue` に上記 2 項目のコントロールを追加
- 既存の粒度 / カスタム指示 / 検索設定と縦並びで自然に並ぶ範囲
- 開発/本番の表示切り替えは `import.meta.dev` 相当の Nuxt 標準判定で行う（最終的な判定方法は IMPLEMENT で決定）

## スコープ外（明示）

- 設定パネル UI の全面刷新
- スレッド間で設定を複製する機能
- MASTER_PLAN PHASE3 項目3（会話分岐）— 次メジャー ver17.0 で扱う
- ISSUES `getState-timing.md` / `additional-kwargs-sqlite.md` の検証 — 手動デプロイ検証の範囲、本バージョンでは対象外

## 前提条件

**必須（ブロッカー）:**

- 既存の `ThreadSettings` JSON スキーマへの項目追加方式が ver15.0〜ver16.0 で安定運用されていること（確認済み: `thread-store.ts` / `settings.put.ts` / `useSettings.ts`）

**推奨（進行可能だが確認が望ましい）:**

- エコーモードでのトークン分割ポリシー（文字数 / 単語 / 固定チャンク）は実装時に 1 パターンに絞る
