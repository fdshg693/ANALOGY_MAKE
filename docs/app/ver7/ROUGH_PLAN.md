# ver7: 自動テスト導入

## 対応する課題

`ISSUES/high/auto-test.md` — 自動テスト導入（高優先度ISSUES対応）

## 概要

現状、動作確認はすべて手動（型チェック・ビルド・ブラウザ確認）に依存しており、リグレッション検知やリファクタ時の安全網がない。Vitestによるテスト基盤を導入し、バックエンドAPIとユーティリティの自動テストを整備する。

PHASE1.5で予定されている composable 切り出し・応答キャンセル等の大規模リファクタリングの前に、安全網を確保することが目的。

## 提供されるもの

### 1. テスト基盤のセットアップ

- Vitest の導入・設定
- テスト用のOpenAI APIモック（APIキーなしでテスト実行可能）
- `pnpm test` でテスト一括実行

### 2. SSEパーサーの切り出しとユニットテスト

`index.vue` 内に埋め込まれているSSEパース処理を、Vue非依存の純粋関数としてユーティリティに切り出す。これにより単体テストが可能になる。

テスト観点:
- 正常系: token → done の基本フロー
- バッファ分割: SSEイベントがチャンク境界で分断されるケース
- エラー系: サーバーからの error イベント受信
- 空レスポンス・不正フォーマットへの耐性

### 3. サーバーAPIハンドラのテスト

`chat.post.ts` のバリデーションとSSEイベントストリーム形式を検証する。

テスト観点:
- バリデーション: message / threadId 欠落時の400エラー
- 正常系: モックされたエージェントからのSSEストリーム形式
- エラー系: エージェント呼び出し失敗時の error イベント送信

## スコープ外

- フロントエンドコンポーネントテスト（E2Eテスト等は将来対応）
- `useChat` composable の切り出し（ver7ではSSEパーサーのみ切り出し、composable化は後続バージョンで対応）
- `useChat` composable のテスト（PHASE1.5で予定されている `useChat.test.ts` は composable 切り出し後の後続バージョンで対応）
- 応答キャンセル機能（PHASE1.5の別タスク）
- ストリーミング表示改善（PHASE1.5の別タスク）

## 備考

- Vitest セットアップ時に `happy-dom` も導入しておく（ver7のテスト対象はDOM不要だが、後続バージョンで `useChat.test.ts` 等のコンポーネントテスト追加時に必要になるため）
