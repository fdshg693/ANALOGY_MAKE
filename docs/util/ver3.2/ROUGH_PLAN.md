## 対応する ISSUE

- `ISSUES/util/medium/ユーザーFB.md`

## 背景

現状、AI のレトロスペクティブや `REQUESTS/AI/` の出力に対して、ユーザーが手動で SKILL ファイルの編集や設定変更を行っている。この手動介入を減らすため、ユーザーはフィードバック内容だけをファイルに記述し、ワークフロー実行時に自動的に AI のプロンプトへ注入する仕組みを導入する。

## スコープ

### やること

- ユーザーがフィードバックを記述するファイルの仕組みを設計する
  - ファイル内で「どのワークフローステップで読み込むか」を指定できる
  - 複数の FB ファイルが同一ステップに紐づく場合も対応する
  - FB ファイルのフォーマット（ステップ指定方法: ファイル名規則 / frontmatter 等）は IMPLEMENT.md で詳細設計する
- `claude_loop.py` がステップ実行前にマッチする FB ファイルを検出し、プロンプトに注入する
  - full / quick 両ワークフローで動作する（ステップ名ベースのマッチングのため、ワークフロー種別に依存しない）
- 読み込み済み FB の管理: 注入後にファイルを消費済みとして扱う仕組みを設ける（具体的な方式は IMPLEMENT.md で決定）

### やらないこと

- SKILL ファイルへの FB 文言の直接埋め込み（ISSUE の制約: メンテナンス性・カスタマイズ性の低下を回避）
- FB の自動生成・自動分類
- FB ファイルのフォーマットバリデーション
- YAML ワークフロー定義の構造変更

## ユーザー体験の変化

**Before**: レトロスペクティブ等の出力を読む → 手動で SKILL ファイルや設定を編集 → 次回ワークフロー実行で反映

**After**: 所定のファイルにフィードバックを記述し、対象ステップ名を指定 → 次回ワークフロー実行時に自動的に AI のプロンプトへ注入 → AI がフィードバックを踏まえて行動
