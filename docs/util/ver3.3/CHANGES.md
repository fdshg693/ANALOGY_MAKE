## 変更内容の詳細

### write_current SKILL: CURRENT.md 分割ルールの具体化

旧記述では「肥大化が考えられる場合は `docs/{カテゴリ}` フォルダ配下に、機能単位のドキュメントなどを作成して、記載を分割することを検討する」という曖昧な1行のみだった。これを削除し、新たに「CURRENT.md のファイル分割」セクションを追加:

- **分割基準**: 150行超でトピック単位に分割、150行以下なら単一ファイル
- **命名規則**: `CURRENT_{トピック名}.md`（バージョンフォルダ内に配置）
- **インデックス構成**: 分割時の `CURRENT.md` は全体概要 + 各詳細ファイルへのリンクで構成
- **各詳細ファイル**: 50〜200行を目安

また、CLAUDE.md との重複排除ルールの記述を簡潔化した（冗長な括弧内の説明を削除）。

### 読み込み側 SKILL 3件: 分割 CURRENT.md の選択的読み込み対応

`split_plan`・`imple_plan`・`quick_plan` の各 SKILL で、CURRENT.md 参照手順に以下の指示を追加:

> `CURRENT.md` が分割されている場合（`CURRENT_{トピック名}.md` へのリンクを含む場合）は、今回のタスクに関連する詳細ファイルのみを読む

これにより、CURRENT.md が分割されていても全ファイルを読む必要がなくなり、コンテキストの効率的な利用が可能になった。

## 技術的判断

- **分割基準 150行**: util カテゴリの ver3.0 CURRENT.md が 297行であったことから、その約半分を基準とした。頻繁な分割を避けつつ、可読性を保つバランスを意図
- **バージョンフォルダ内配置**: 旧記述の `docs/{カテゴリ}` 直下ではなく、`docs/{カテゴリ}/ver{X.0}/` 内に `CURRENT_{トピック名}.md` を配置する方式を採用。バージョンごとの自己完結性を維持するため
