# ver10: 会話履歴の永続化（SQLite導入）

## 背景

PHASE1.5 は実質完了（composable切り出し・応答キャンセル・テスト導入済み、ストリーミング表示改善はlow ISSUE として管理中）。
PHASE2 の最初のステップとして、会話履歴の永続化に着手する。

## 対応内容

PHASE2 の 4項目のうち、**「1. 会話履歴の永続化（SQLite）」** のみを対応する。

### 提供される機能

- サーバー側に SQLite データベースを導入し、LangGraph の会話チェックポイントを永続化する
- 現在のインメモリ `MemorySaver` を SQLite ベースのチェックポインターに置き換える
- サーバー再起動後も会話を復元できるようになる
- ブラウザのページリロード後も同じ会話を継続できる（`threadId` を `localStorage` に保持）

### 技術方針

- SQLite チェックポインター: `@langchain/langgraph-checkpoint-sqlite` の利用を第一候補とする。利用不可の場合は `better-sqlite3` + `BaseCheckpointSaver` 継承による自前実装を検討
- `threadId` の永続化: クライアント側で `localStorage` に保存し、ページリロード時に復元する

### スコープ

- バックエンドのチェックポインター差し替え
- フロントエンドは `threadId` の `localStorage` 保持のみ
- マルチスレッドUI（サイドバー・スレッド一覧）は含まない（ver11 以降）
- Web検索連携・ステートマシン化は含まない

### ユーザー体験の変化

- **変更前**: サーバー再起動またはページリロードで会話履歴が消失
- **変更後**: サーバー再起動・ページリロード後も会話が継続可能

## リファクタリング

事前リファクタリング不要（MemorySaver→SqliteSaver はドロップイン置換、threadId 永続化も useChat 内の小変更のみ。REFACTOR.md 省略）

## 後続タスクとの関係

本タスクは PHASE2 の基盤となる。後続バージョンで以下を順次対応予定:
- ver11（予定）: 複数スレッド管理UI
- 以降: Web検索連携、ステートマシン化
