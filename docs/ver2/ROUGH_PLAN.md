# Task 2: LangChain + Node.js 実験フェーズ

## 概要

Nuxtアプリへの組み込みに先立ち、**LangChain + OpenAI APIをNode.jsスクリプトとして単体で動かし**、アナロジー思考フローに必要な技術要素を検証する。
最新のLangChain.js ドキュメントを確認し、会話メモリ管理・プロンプト設計の実装方法を確かめた上で、直接実行できるスクリプトとして動作確認を行う。

## 提供する機能

### 検証用スクリプト群（`experiments/` ディレクトリ）

直接 `npx tsx experiments/xxx.ts` で実行できるスタンドアロンスクリプト：

1. **基本接続確認** — LangChainからOpenAI APIを呼び出し、応答が返ることを確認
2. **会話メモリ管理** — 会話履歴の保持・受け渡し方法を検証（LangChainのメモリ機構 or 手動管理）
3. **アナロジー思考プロンプト** — 5ステップフローを実現するプロンプト設計の検証

### ドキュメント調査

- LangChain.js の最新API（v0.3系）のメモリ・チャットモデル周りの確認
- 破壊的変更や非推奨APIの把握
- Node.js環境での推奨構成の確認

## 検証観点

| 観点 | 確認すること |
|---|---|
| API接続 | LangChainからOpenAI APIを呼べるか、レスポンス形式はどうか |
| メモリ管理 | 会話履歴をどう保持し、次のリクエストに渡すか |
| プロンプト設計 | システムプロンプトで5ステップフローをどこまで制御できるか |
| エラー挙動 | APIキー未設定・レート制限時にどういうエラーが返るか |

## 完了条件

- `experiments/` 配下に動作確認済みのスクリプトが存在する
- 各スクリプトが `npx tsx` で単体実行でき、期待通りの応答を得られる
- 会話メモリの管理方法が確定し、コード上で実証されている
- アナロジー思考の5ステップフローが対話として成立することがプロンプトレベルで確認できている
- 検証結果が次タスク（Nuxtアプリへの組み込み）の参考実装として使える状態になっている
