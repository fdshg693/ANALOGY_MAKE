# ver5 タスク概要: SSEエラーハンドリングとエラーUI改善

## 概要

サーバーは `token` / `done` / `error` の3種のSSEイベントを送信するが、クライアント (`index.vue`) は `token` イベントのみ処理している。ストリーム途中のエラー（OpenAI APIレート制限、ネットワーク障害など）がユーザーに通知されず、無言でストリームが終了する。本バージョンでは、SSEの全イベントを適切にクライアントで処理し、エラー発生時にユーザーへわかりやすいフィードバックを提供する。

## 現状の課題

- `error` イベントが送信されても、クライアントは無視してストリーム終了を待つだけ
- `done` イベントも明示的に処理されておらず、ストリーム完了の確定が `ReadableStream` の終了に依存している
- エラー発生時、ユーザーには原因が伝わらず「応答を取得できませんでした」としか表示されない
- ストリーミング途中でエラーが起きた場合、途中まで表示されたテキストの扱いが不明確

## 提供される機能

### 1. SSE `error` イベントのクライアント処理

- クライアントのSSEパーサーで `error` イベントを検知する
- エラー発生時にストリーミングを即座に停止し、適切な状態リセットを行う

### 2. SSE `done` イベントの明示的処理

- `done` イベントを受信した時点でストリーム完了を確定する
- `ReadableStream` の終了だけに依存しない、より堅牢な完了判定

### 3. エラーメッセージのUI表示

- エラー内容をチャットUI上でユーザーに表示する（assistantメッセージ内にエラー表示、またはエラー専用の表示領域）
- ストリーミング途中でエラーが発生した場合、途中テキストを保持しつつエラーを付記する
- エラー後も会話を継続できる状態にする（入力欄の再有効化）

## 完了条件

- `error` イベント受信時にエラーメッセージがUI上に表示される
- `done` イベントが明示的に処理され、ストリーム完了が確定される
- ストリーミング途中のエラーでも、途中テキストが保持される
- エラー後に入力欄が再有効化され、会話を継続できる
- 既存の正常系ストリーミング動作に影響がない

## スコープ外（本タスクでは対応しない）

- Markdownレンダリング
- 応答の中断・キャンセル機能
- 自動テストの導入（別ISSUE）
- SSEパーサーのユーティリティ切り出し（低優先度ISSUE）
- composable切り出し（低優先度ISSUE）
