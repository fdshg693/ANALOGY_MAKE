# ver6 リファクタリング計画

## 結論: リファクタリング不要

本タスク（Markdownレンダリング導入）に先立つリファクタリングは不要。

### 理由

- 変更対象は `ChatMessage.vue`（55行）がメインで、コンポーネントが十分に小さく、変更を直接加えられる
- props インターフェース（`role`, `content`, `isError`）は変更不要。親コンポーネント（`index.vue`）の修正は不要
- `white-space: pre-wrap` から `v-html` + Markdown スタイルへの切り替えは、同コンポーネント内のテンプレートとスタイルの変更で完結する
- ストリーミングのリアクティビティ（`content` prop の逐次更新）は既存の仕組みがそのまま活用可能
